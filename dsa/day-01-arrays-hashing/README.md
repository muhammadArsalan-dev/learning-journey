**Track:** Big Tech — Day 1 of 90

## Problems Solved
- Contains Duplicate — used a Set for O(1) existence checks
- Valid Anagram — built frequency maps for both strings, compared key by key
- Two Sum — hash map storing {value: index}, checked for complement before storing

## What I Learned
- I learned how to leverage hash maps and sets to trade spatial complexity for optimal time complexity, transforming nested loop lookups into efficient single-pass solutions.

## Complexity

- Valid Anagram: Time O(n), Space O(n)
- Two Sum: Time O(n), Space O(n)
